Hello shibe!

Welcome to DogeTips, an easy way to integrate Dogecoins into your Minecraft server!

Let's take a look at what this plugin currently has to offer...

What are the commands?

There's only one command that you will have to remember:! '/doge' contains all the nifty sub-commands your players may use to interact with DogeTips. You can use a sub-command with '/doge (sub-command)'

Alright, what are these 'sub-commands'?

Glad you asked! Here are the current sub-commands and their usage (substitute "you" with whoever is sending this command):

  • /doge help <page number>: Informs you about this plugin and how to use it.
  • /doge deposit: Will give you an address that you should deposit coins to. Use this to add coins into your Minecraft wallet.
  • /doge withdraw <amount> <address>: Will remove your in-game Dogecoins and send them to the specified address!
  • /doge check: Will force the game to check and see if your Dogecoin deposit transaction went through. Be patient as this may take some time depending on the speed of the block chain :)
  • /doge jar: Provides a statement informing you of your stored Dogecoins.
  • /doge tip <player> <amount>: Last, but certainly not least: tip other players in Dogecoin!

I have no idea how to do this. Help me?

Fear not! It's not that difficult if we take it step by step.
1. Download the latest version of DogeTips from the Bukkit page.
2. You should now have a .jar file. Drag 'n drop this into your server's /plugins folder.
3. Start up your server and then stop it. You may also /reload the server if you wish (be careful as reloading is known to cause errors in some other plugins).
3. Go back into your /plugins folder. Double-click on the new folder entitled DogeTips.
4. Open the file config.yml into your favorite text editor. Move onto the section titled Let's configure! (config.yml).

Let's configure! (config.yml)

When you open the config.yml for the first time you should get something similar to below. If you ever make a huge mistake with the config, you can just delete it and reload your server to generate a new one. 

    #Set this to true if you wish to integrate DogeTips with an existing economy plugin that uses Vault.
    use-vault-economy: false

    #Ignored if use-vault-economy is true.
    separate-economy:
      balance-file-name: 'balances.csv'

    #API configuration.
    api:
      api-key: "YOUR_API_KEY_HERE"
      base-url: "https://block.io/api/v1/"
      api-key-query: "/?api_key="

    #Plugin colors. Use an ampersand for formatting codes if you wish.
    colors:
      primary: '&e'
      secondary: '&7'
      errors: '&c'

Nodes (or everything ending in a colon):

use-vault-economy: This must be true or false. If true, DogeTips will use an existing economy that is already on your server. The Vault plugin must be in your plugins folder if you enable this. If this is false, then DogeTips will use its own economy that does not need Vault.
balance-file-name: This is the file name that is used to store Dogecoin balances for all players if the node use-vault-economy is set to false.
api-key: Replace this with your API key found on Block.io.
base-url and api-key-query: Do not worry about these for now. In the case that block.io removes v1 support, this URL should be changed.
colors: You can change the colors of the plugin's messages. It uses the standard Minecraft color code system.
